Code 51: Semantic Layout

Title, meta, content & sidebar - HTML5 semantic tags only

What it does

This layout uses a Main-Sidebar structure with proper HTML5 semantic elements. By separating metadata from body content, we improve scannability and SEO.

The foundation for blogs, docs, and articles. Uses <article>, <aside>, <nav> correctly.

Understanding Semantic Layouts

Published: June 15, 2026 | Category: CSS Architecture

This layout demonstrates proper document structure. The <main> element contains the primary content, while <aside> holds complementary info like related links.

Using semantic tags helps screen readers, SEO crawlers, and future developers understand your document hierarchy without classes.

The Code

<div class="main-wrapper">
  <main class="primary-content">
    <article>
      <header><h2>Title</h2></header>
      <section>Body content</section>
    </article>
  </main>
  <aside class="sidebar">
    <nav>...</nav>
  </aside>
</div>
.main-wrapper {
  display: grid;
  grid-template-columns: 1fr 300px;
  gap: 2rem;
  align-items: start;
}

@media (max-width: 768px) {
  .main-wrapper {
    grid-template-columns: 1fr;
  }
}

Key Rules

  • Use <main> once per page for primary content — skip navs, sidebars, headers
  • <article> for self-contained content that makes sense alone: blog post, comment
  • <aside> for tangentially related content: sidebars, callouts, ads
  • <nav> only for major navigation blocks, not every link group
  • grid-template-columns: 1fr 300px creates content + fixed sidebar
  • align-items: start prevents sidebar stretching to match article height
  • Stack with grid-template-columns: 1fr at 768px breakpoint
  • Meta data in <header> inside <article> improves structure
