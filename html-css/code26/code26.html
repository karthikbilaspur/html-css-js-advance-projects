Code 26: Accordion FAQ

Disclosure widget with <details>/<summary> - zero JavaScript

What it does

The <details> element creates a disclosure widget that users can open and close. Perfect for FAQs — no JavaScript needed.

The <summary> is the visible question. Everything else inside <details> is the answer shown when open.

Live Demo

What is semantic HTML?

Semantic HTML uses tags that describe their meaning, like <header>, <nav>, <article>, and <footer>.

This helps screen readers, search engines, and developers understand your page structure better than generic <div> tags.

Do I need JavaScript for dropdowns?

Not always. You can use :hover and :focus-within for basic dropdowns, or <details> for click-to-open menus.

JavaScript is only needed for complex interactions like closing on outside click or keyboard Esc key support.

Can multiple details be open at once?

Yes, by default each <details> works independently. A user can open all FAQs at once.

If you want only one open at a time, use the name attribute: <details name="faq"> groups them.

How do I style the default arrow?

Target summary::marker or summary::-webkit-details-marker and set display: none.

Then add your own icon with ::after on the summary and rotate it when details[open].

↑ Click any question to expand. Multiple can be open at once.

The Code

<details class="faq-item">
  <summary>What is HTML?</summary>
  <div class="answer">
    <p>HTML stands for HyperText Markup Language...</p>
  </div>
</details>
/* Custom arrow icon */
details summary::after {
  content: "+";
  float: right;
}

details[open] summary::after {
  content: "−";
}

Key Rules

  • <summary> must be the first child of <details>
  • Remove default marker with summary::marker { display: none; } or list-style: none
  • Style open state with details[open] summary
  • Add name="group" to multiple <details> to make them mutually exclusive
  • Keyboard accessible by default — Tab to focus, Space/Enter to toggle
  • Use cursor: pointer on summary to show it's clickable
