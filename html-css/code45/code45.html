Code 45: Stats Counter

Animated numbers with @property - zero JavaScript

What it does

This creates a CSS-only number counter. By using @keyframes to animate a CSS variable defined with @property, we force the browser to recalculate and display increasing numbers.

Pure CSS animation. No JS setInterval. GPU accelerated.

The Code

<div class="stat-item">
  <span class="counter" style="--num: 150"></span>
  <p>Projects</p>
</div>
@property --count {
  syntax: '<integer>';
  initial-value: 0;
  inherits: false;
}

.counter {
  animation: count 2s forwards;
  counter-reset: num var(--count);
}

.counter::after {
  content: counter(num) '+';
}

@keyframes count {
  to { --count: var(--num); }
}

Key Rules

  • @property --count defines custom property as <integer> so it can be animated
  • counter-reset: num var(--count) ties CSS counter to animated variable
  • content: counter(num) renders the current integer value
  • animation-fill-mode: forwards keeps counter at final number
  • Set target with inline style: style="--num: 150"
  • @keyframes animates --count from 0 to --num
  • Fallback: Use static numbers if @property unsupported in old browsers
