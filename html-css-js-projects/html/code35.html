Code 35: <dt> Description Term

The <dt> element specifies a term in a description list <dl>. It must come before its <dd> description.

What it does

<dt> is the "name" in a name-value pair. It’s the thing being defined.

Without <dl> as a parent, <dt> is invalid. It always pairs with <dd>.


The Code

HTML 
<dl>
  <dt>HTML</dt>
  <dd>The standard markup language for web pages.</dd>
</dl>

FAQ pattern

What is semantic HTML?
HTML that uses elements for their meaning, not just appearance.
Why use description lists?
For glossaries, metadata, and any term-description pairs.

dt Rules

dt rules: <dt> must be inside <dl> and must come before <dd>.

It represents the term being described. You can group multiple <dt> elements if they share the same definition.

It should be inline content only — no blocks like <p> or <div> inside. Keep it to text, <abbr>, <code>, etc.
