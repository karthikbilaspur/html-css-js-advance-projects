Code 31: <ul> Unordered List

The <ul> element represents an unordered list of items. Order doesn't matter — usually rendered with bullets.

What it does

<ul> groups items where sequence doesn’t matter: features, ingredients, nav links.

Only <li> can be direct children. Bullets are just the browser default — meaning is semantic.


The Code

HTML 
<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>

ul vs ol

ul vs ol: Use <ul> when order doesn't matter. Use <ol> for numbered steps or rankings.

Only <li> elements can be direct children of <ul>. Browsers default to disc bullets, but that's just presentation.

Pro tip: That nav example? That’s exactly how 90% of site menus are built. <nav><ul><li><a> is the pattern.
