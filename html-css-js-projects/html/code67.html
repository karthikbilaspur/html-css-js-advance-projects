Code 67: <legend>

The <legend> element provides a caption for the parent <fieldset>. It should be the first child and describes the grouped controls.

Note

legend position: Must be the first child of <fieldset>. If placed later, browsers still render it at the top but it’s invalid HTML.

Accessibility requirement: For radio and checkbox groups, <legend> is how you tell assistive tech what the group is about. Without it, users hear "Option 1" with no context.

Styling: Browsers place legend text on the fieldset border. You can style it but it behaves different from normal text — it’s part of the border rendering.

Content rules: Use phrasing content only. No <div>, <p>, or headings inside. Use <span>, <em>, <strong> if needed.
