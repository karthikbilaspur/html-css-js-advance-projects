Code 95: <style>

The <style> element contains internal CSS for the document. It goes in <head> and applies styles to the current page only.

Examples

1. Basic internal stylesheet

<head>
  <style>
    body { font-family: Arial; }
    h1 { color: blue; }
  </style>
</head>
CSS rules inside <style> apply to the whole page

2. Scoped to media type

<style media="print">
  nav, footer { display: none; }
  body { color: black; }
</style>
Styles only apply when printing. Hide nav/footer on paper

3. Multiple style blocks

<style>
  /* Critical above-the-fold CSS */
  .hero { height: 100vh; }
</style>

<link rel="stylesheet" href="main.css">

<style>
  /* Page-specific overrides */
  .hero { background: url(special.jpg); }
</style>
You can mix internal <style> with external <link>

4. CSS with nonce for CSP

<style nonce="r4nd0m">
  .secure { display: block; }
</style>
Used with Content-Security-Policy to allow only approved inline styles

Warning: Internal CSS can't be cached across pages like external .css files.

Note

style purpose: <style> lets you write CSS directly in HTML. Good for single-page sites, email templates, or critical CSS.

Location: Always goes in <head>. Browsers allow it in <body> but it’s invalid HTML and hurts performance.

Internal vs External vs Inline:

Type Syntax Use case
External <link rel="stylesheet"> Multi-page sites, cacheable
Internal <style>...</style> Single page, critical CSS
Inline style="color:red" Quick overrides, avoid normally

Performance: Internal CSS loads with HTML - no extra request. But it blocks rendering and isn't cached. External CSS is cached after first load.

Critical CSS pattern: Put above-the-fold styles in <style> for fast first paint, then load rest with <link>.

No scoped attribute: HTML5 removed scoped. All <style> rules apply globally to the document.

CSP issues: Content-Security-Policy often blocks inline <style>. Use nonce or hash, or prefer external files.
