Code 86: <section>

The <section> element represents a thematic grouping of content. Each section should have a heading. Use it to divide a page into topics.

Note

section purpose: Groups related content by theme. Each <section> should typically have a heading <h1>–<h6>.

section vs article: <section> = thematic grouping. <article> = standalone content. If it can't stand alone, use section.

section vs div: <div> has no meaning — just a container. <section> tells browsers and screen readers "this is a distinct topic".

Heading rule: Sections should have headings. If no heading makes sense, <div> might be better. Or add aria-label.

Don't overuse: Not every group needs <section>. Use when you have a logical topic boundary with a heading.
