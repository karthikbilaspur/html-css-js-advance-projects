Code 96: <script>

The <script> element embeds or references executable code, typically JavaScript. It can go in <head> or <body>.

Examples

1. External JavaScript file

<script src="app.js"></script>
Most common. Loads JS from a file. Put before </body> or use defer

2. Inline JavaScript

<script>
  console.log('Hello from inline JS!');
  document.getElementById('btn').onclick = () => alert('Clicked');
</script>
Code runs immediately. Keep it small - big scripts belong in files

3. defer - load after HTML parsing

<script src="app.js" defer></script>
Downloads in parallel, executes after HTML is parsed. Best for scripts that need the DOM

4. async - load independently

<script src="analytics.js" async></script>
Downloads in parallel, executes ASAP when ready. Use for independent scripts like analytics

5. ES Modules

<script type="module">
  import { sum } from './utils.js';
  console.log(sum(2, 3));
</script>
Modern JS. Supports import/export. Modules are deferred by default

6. External module

<script type="module" src="main.js"></script>
Loads main.js as an ES module. Can use imports inside

7. No JavaScript fallback

<script src="app.js"></script>
<noscript>
  <p>This app requires JavaScript to run.</p>
</noscript>
<noscript> shows only when JS is disabled

8. JSON-LD structured data

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "HTML Script Tag Guide"
}
</script>
Not executed as JS. Used for SEO structured data

9. defer vs async vs normal

AttributeDownloadExecuteUse for
noneBlocks HTMLImmediatelySmall critical inline
deferParallelAfter HTMLScripts needing DOM
asyncParallelASAPAnalytics, ads

Note

script purpose: <script> loads or defines executable code. Mostly JavaScript, but can hold JSON, shaders, or templates via type.

Blocking by default: Normal <script> blocks HTML parsing until it downloads + runs. This hurts page speed.

defer is your friend: Use defer for your own scripts. It won’t block rendering and runs in order after DOM is ready.

async for independent code: Use async for analytics, ads, or tracking. Order isn’t guaranteed.

Placement: Old rule: scripts at bottom of <body>. Modern rule: use defer in <head>.

type="module": Enables ES6 imports. Modules are automatically deferred and run in strict mode. Always use .js extension in imports.

No src + content: If you set src, any content inside the tag is ignored.

Security: Never inline user data. Use Content-Security-Policy and avoid eval(). Consider integrity and crossorigin for CDN scripts.
