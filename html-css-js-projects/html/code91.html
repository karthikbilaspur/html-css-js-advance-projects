Code 91: <meta>

The <meta> element provides metadata about the HTML document. It always goes in <head> and is not displayed on the page.

Examples

1. Character encoding - always first

<meta charset="UTF-8">
Defines character set. UTF-8 handles all languages + emoji. Put this first in <head>

2. Viewport for responsive design

<meta name="viewport" content="width=device-width, initial-scale=1.0">
Makes your site mobile-friendly. Without it, phones show zoomed-out desktop view

3. SEO description

<meta name="description" content="Learn HTML in 100 code examples">
Shows up in Google search results. Keep under 160 characters

4. Prevent search indexing

<meta name="robots" content="noindex, nofollow">
Tells search engines not to show this page in results. Use for admin pages

5. Page refresh or redirect

<meta http-equiv="refresh" content="5; url=https://example.com">
Redirects after 5 seconds. Avoid for UX - use server redirects instead

6. Theme color for mobile browsers

<meta name="theme-color" content="#1a73e8">
Colors the browser address bar on Android Chrome

7. Open Graph for social sharing

<meta property="og:title" content="My Article">
<meta property="og:image" content="https://site.com/image.jpg">
Controls how your link looks when shared on Facebook, LinkedIn, WhatsApp

8. Twitter Cards

<meta name="twitter:card" content="summary_large_image">
<meta name="twitter:title" content="My Article">
Twitter-specific tags for rich previews. Falls back to og: tags if missing

Note

meta purpose: <meta> tags describe your page to browsers, search engines, and social platforms. They never show in page content.

Required metas: Every page should have charset and viewport. For SEO, add description.

name vs property vs http-equiv: Use name for standard metas, property for Open Graph, http-equiv for HTTP header equivalents.

No closing tag: <meta> is void. Never write </meta>.

Order matters: charset must be within first 1024 bytes. Put it right after <head>.

Too many metas: Only add what you need. Bloated <head> slows initial page load.

Case sensitive: Attribute values like UTF-8 are case-insensitive, but be consistent.
