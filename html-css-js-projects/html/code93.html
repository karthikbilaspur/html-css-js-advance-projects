Code 93: <meta name="author">

The author meta tag specifies who wrote the page content. It goes in <head> and is mainly used for documentation, blogs, and articles.

Examples

1. Basic individual author

<meta name="author" content="Alex Chen">
Use full name for personal blogs, articles, or portfolios

2. Organization as author

<meta name="author" content="Mozilla Developer Network">
For company docs, wikis, or team-written content

3. Multiple authors

<meta name="author" content="Priya Sharma, Rahul Verma">
Comma-separated list. Order usually reflects contribution level

4. Blog post with author

<head>
  <title>CSS Grid vs Flexbox</title>
  <meta name="author" content="Sarah Johnson">
  <meta name="description" content="When to use Grid vs Flexbox...">
</head>

5. Email as author

<meta name="author" content="contact@example.com">
Less common now. Spammers scrape these, so avoid personal emails

6. Author + copyright together

<meta name="author" content="HTML 100 Codes Vault">
<meta name="copyright" content="© 2026 HTML 100 Codes Vault">
Author = who made it, copyright = who owns it

7. Don't do this

Bad - keyword stuffing:

<meta name="author" content="HTML tutorial, learn HTML, HTML expert, coding">
Author tag is for names, not keywords

Note

author purpose: <meta name="author"> identifies the content creator. It’s metadata, not displayed on the page.

SEO impact: Zero direct ranking benefit. Google doesn’t use it for rankings. But it helps with documentation, CMS tools, and internal audits.

When to use it: Blogs, articles, research papers, documentation. Skip it for homepages, product listings, or generic pages.

vs byline: This meta tag is for machines. For humans reading the page, still show the author visibly in an <address> or byline.

Format: Just the name. No "By" or "Written by". Example: content="Jane Doe" not content="By Jane Doe".

Multiple pages, same author: It’s fine to repeat the same author tag across your site if you wrote all content.

Not required: Most sites skip this tag entirely. It’s optional and rarely used in modern SEO.

Better alternatives: For author info that matters to Google, use structured data with schema.org/Person or visible bylines with rel="author".
