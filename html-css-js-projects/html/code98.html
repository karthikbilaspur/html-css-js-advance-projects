Code 98: <base>

The <base> element specifies the base URL and default target for all relative URLs in a document. Only one <base> per page, and it must be in <head>.

Examples

1. Base URL for all relative links

<head>
  <base href="https://example.com/tutorials/">
</head>

<body>
  <!-- This points to https://example.com/tutorials/html.html -->
  <a href="html.html">HTML Tutorial</a>
  
  <!-- This points to https://example.com/tutorials/css.html -->
  <a href="css.html">CSS Tutorial</a>
</body>
All relative href and src resolve against the base URL

2. Base target for links

<base target="_blank">

<a href="https://google.com">Google</a>
<a href="/contact">Contact</a>
Both links open in new tab by default. Override with target="_self" on specific links

3. Base href + target combined

<base href="https://cdn.example.com/assets/" target="_blank">

<img src="logo.png">
<!-- Loads from https://cdn.example.com/assets/logo.png -->

<a href="doc.pdf">PDF</a>
<!-- Opens https://cdn.example.com/assets/doc.pdf in new tab -->

4. How it affects different elements

  • <a href="page.html"> → uses base
  • <img src="pic.jpg"> → uses base
  • <link href="style.css"> → uses base
  • <script src="app.js"> → uses base
  • <form action="submit"> → uses base

5. Absolute URLs ignore base

<base href="https://example.com/docs/">

<a href="https://google.com">Google</a>
<!-- Still goes to google.com, not example.com/docs/google.com -->

<a href="/about">About</a>
<!-- Goes to https://example.com/about, not /docs/about -->
Absolute URLs and root-relative URLs starting with / ignore <base>

6. Common gotcha with anchors

<base href="https://example.com/app/">

<a href="#section">Jump</a>
<!-- Goes to https://example.com/app/#section -->
<!-- Not the current page! Use href="" to stay on page -->
Warning: <base> breaks same-page anchors. href="#id" will jump to base URL + #id

7. When NOT to use base

Avoid if:

  • Your site uses client-side routing like React Router
  • You have in-page anchors like #top, #footer
  • Multiple developers work on the same codebase - it’s confusing

Note

base purpose: <base> sets a document-wide default URL and browsing context for all relative URLs.

Only one allowed: If multiple <base> tags exist, only the first href and first target are used.

Must be in head: Must appear before any elements with URL attributes like <link>, <img>, or <a>.

href must be absolute: href="/folder/" or href="https://...". Relative base hrefs are ignored.

target values: _self, _blank, _parent, _top, or a named frame. _blank is most common.

Breaks anchors: Biggest downside. <a href="#section"> jumps to baseURL#section instead of current page. Fix with href="current-page.html#section".

Modern usage: Rarely used today. SPAs and build tools handle paths better. Can still help with static sites hosted in subdirectories.
