Code 29: Sticky Header on Scroll

What it does

position: sticky makes an element stick to the top when you scroll past it. Unlike position: fixed, it stays in the document flow until it hits the scroll threshold.

Perfect for headers that should remain visible but not overlap content initially. Works with top: 0 to stick to viewport top.

The Code

<header class="sticky-header">
  <nav>...</nav>
</header>
.sticky-header {
  position: sticky;
  top: 0;
  z-index: 100;
  background: #fff;
  box-shadow: 0 2px 8px rgba(0,0,0,0.1);
}

Key Rules

  • Parent must NOT have overflow: hidden or overflow: auto — breaks sticky
  • Must set top, bottom, left, or right value. Use top: 0 for header
  • Add z-index so it stays above scrolling content
  • Add background color — otherwise content shows through when stuck
  • position: sticky keeps element in normal flow until scroll threshold
  • Unlike fixed, it won't overlap content that comes before it
  • Add shadow or border when stuck for visual separation: box-shadow
