Code 27: Pagination UI

What it does

Pagination splits content across multiple pages. This component uses <nav aria-label="pagination"> with numbered links and shows the current page with aria-current="page".

The active page is styled differently and disabled to prevent clicking yourself. Prev/Next buttons handle navigation.

The Code

<nav aria-label="Pagination">
  <a href="#prev">‹ Prev</a>
  <ul>
    <li><a href="#1">1</a></li>
    <li><a href="#2" aria-current="page">2</a></li>
    <li><a href="#3">3</a></li>
  </ul>
  <a href="#next">Next ›</a>
</nav>
.page-link[aria-current="page"] {
  background: #4dabf7;
  color: #fff;
  pointer-events: none;
}

Key Rules

  • Wrap in <nav aria-label="Pagination"> for screen readers
  • Use <ul> for page numbers because it's a list of links
  • Mark active page with aria-current="page" not just a class
  • Disable active link with pointer-events: none so it can't be clicked
  • Add aria-label to Prev/Next if they only show icons
  • Use aria-disabled="true" on Prev when on page 1
  • Add ellipsis … for skipped pages in long paginations
