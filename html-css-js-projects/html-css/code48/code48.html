Code 48: Newsletter Section

Email capture with flex inline layout - zero JavaScript

What it does

This layout provides a clean, user-friendly interface for capturing email addresses. It ensures the input and button remain inline on desktop and stack gracefully on mobile.

Essential for lead generation, blog subscriptions, and product updates.

The Code

<form class="newsletter-form">
  <label for="email" class="sr-only">Email</label>
  <input type="email" id="email" placeholder="Enter your email" required>
  <button type="submit">Join</button>
</form>
.newsletter-form {
  display: flex;
  gap: 0.5rem;
  max-width: 500px;
}

.newsletter-form input {
  flex-grow: 1;
  padding: 0.75rem 1rem;
  border: 1px solid #ccc;
  border-radius: 6px;
}

.newsletter-form button {
  padding: 0.75rem 1.5rem;
  border: none;
  border-radius: 6px;
  white-space: nowrap;
}

@media (max-width: 480px) {
  .newsletter-form {
    flex-direction: column;
  }
}

Key Rules

  • display: flex on form keeps input and button inline
  • flex-grow: 1 on input makes it consume remaining space
  • gap adds spacing without margin hacks or extra wrappers
  • Match padding + border-radius on input/button for visual alignment
  • white-space: nowrap prevents button text wrapping
  • Use @media (max-width: 480px) + flex-direction: column to stack on mobile
  • Add type="email" + required for native validation
  • Use .sr-only label for accessibility without visual clutter
