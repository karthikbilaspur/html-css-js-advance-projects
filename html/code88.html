Code 88: <aside>

The <aside> element represents content that’s tangentially related to the content around it. Think sidebars, pull quotes, ads, or related links.

Examples

aside inside article

Understanding Semantic HTML

Semantic HTML uses elements that clearly describe their meaning to both browsers and developers. This improves accessibility and SEO.

Elements like <header>, <nav>, <main>, and <footer> define page structure.

Quick Tip

Using semantic tags helps screen readers navigate your page. Users can jump directly to <aside> landmarks with keyboard shortcuts.

Choosing the right element makes your HTML more meaningful than just stacking <div> elements everywhere.

Page-level sidebar layout

In layouts, <aside> is commonly used for the entire sidebar column:

<body>
  <main>...</main>
  <aside>Sidebar content</aside>
</body>
aside doesn’t have to be visually on the side — semantic relationship matters

Note

aside purpose: Use <aside> for content related to the main content, but that could be removed without affecting the primary meaning.

Two main uses: 1. Sidebar related to the whole page. 2. Complementary content inside an <article>, like pull quotes or glossary terms.

Not just sidebars: It doesn’t have to be visually on the side. The semantic relationship matters more than positioning.

Multiple asides: You can have several. Use aria-label like "Related links" or "Advertisement" to help screen readers.

vs section: If it’s essential to the main content, use <section>. If it’s bonus/extra info, use <aside>.

Accessibility: <aside> is a landmark role. Screen reader users can jump straight to all asides on a page.
