Code 94: <link>

The <link> element links external resources to your document. It always goes in <head> and is a void element with no closing tag.

Examples

1. External CSS stylesheet

<link rel="stylesheet" href="styles.css">
Most common use. Loads CSS for styling your page

2. Favicon - multiple formats

<link rel="icon" href="/favicon.ico">
<link rel="icon" href="/icon.svg" type="image/svg+xml">
<link rel="apple-touch-icon" href="/apple-touch-icon.png">
Modern approach: .ico fallback + SVG for crisp scaling + Apple touch icon

3. Google Fonts

<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Inter&display=swap" rel="stylesheet">
preconnect speeds up font loading by warming the connection early

4. Canonical URL for SEO

<link rel="canonical" href="https://example.com/blog/html-guide/">
Tells Google which URL is the "main" one if you have duplicate content

5. Preload critical resources

<link rel="preload" href="/hero.jpg" as="image">
<link rel="preload" href="/fonts/main.woff2" as="font" type="font/woff2" crossorigin>
Downloads key files early. Use for LCP images and above-the-fold fonts

6. Alternate language versions

<link rel="alternate" hreflang="es" href="https://example.com/es/code94.html">
<link rel="alternate" hreflang="fr" href="https://example.com/fr/code94.html">
Helps search engines serve the right language version

7. RSS/Atom feed

<link rel="alternate" type="application/rss+xml" title="Blog RSS" href="/feed.xml">
Lets browsers and RSS readers auto-discover your feed

8. Module preload for JS

<link rel="modulepreload" href="/app.js">
Preloads ES modules for faster execution

Note

link purpose: <link> defines relationships between your document and external resources. It never renders content.

Required attributes: rel is mandatory. It defines the relationship type. href is required for most types.

Void element: No closing tag. Never write </link>.

Order matters for CSS: Later stylesheets override earlier ones. Put your main CSS last.

Performance: preconnect and preload are performance hints. Overusing preload can actually slow things down.

rel="stylesheet" blocking: CSS blocks rendering. Put critical CSS inline or use media="print" onload="this.media='all'" tricks for non-critical CSS.

vs script: Use <link> for resources the browser fetches. Use <script> for executable code.

Favicon minimum: At minimum, provide rel="icon" href="/favicon.ico" in root. Browsers look for it automatically.
