Code 23: <address>

The <address> element represents contact information for its nearest <article> or <body> element. Not for arbitrary postal addresses.

What it does

<address> is semantic — it tells browsers "this is how to contact the author/owner."

Browsers usually render it in italics. It can include email, physical address, phone, social links, etc.


The Code

HTML 
<address>
  Written by <a href="mailto:webmaster@example.com">John Doe</a><br>
  Visit us at:<br>
  Example.com<br>
  123 Web Street<br>
  Bangalore, Karnataka 560001<br>
  India
</address>

HTML Best Practices

Always use semantic tags where possible.

Common Mistake

address usage: Use <address> only for contact info of the page or article author/owner.

Don't use it for every postal address on your site — use <p> for those. <address> is about "who to contact", not "where something is."
