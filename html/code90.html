Note

span purpose: <span> is the inline version of <div>. Zero semantic meaning. It’s just a hook for CSS or JS.

When to use it: When you need to style or manipulate a piece of text but there’s no semantic element like <strong>, <em>, <mark>, or <time> that fits.

Inline behavior: Unlike <div>, <span> does not start on a new line. It flows with surrounding text.

Overuse: Don't wrap everything in spans. If the text has meaning, use semantic tags. "Span soup" is just as bad as "div soup".

Accessibility: <span> adds no meaning for screen readers. If you need to convey emphasis, use <em> or <strong> instead.

Common pattern: <span class="sr-only"> is used to add text only for screen readers, hidden visually with CSS.
