Code 70: <progress>

The <progress> element displays the completion progress of a task. Shows determinate or indeterminate state.

Examples

Basic progress with value


70%
70% complete

Progress without value


loading...
Indeterminate state — no value set

Progress with max attribute


3 of 10
3 out of 10 tasks

Progress with JavaScript update


25%

File upload progress


60%
value can be decimal when max="1"

Multiple progress bars

HTML: 90%

CSS: 60%

JS: 30%


Note

progress states: With value = determinate, shows completion %. Without value = indeterminate, shows activity but no %.

Key attributes: value is current amount. max is total amount, defaults to 1 if omitted. Both take numbers only.

Content fallback: Text inside <progress>70%</progress> shows in old browsers and for screen readers. Always include it.

progress vs meter: Use <progress> for task completion. Use <meter> for static gauges like disk usage or grades.

Accessibility: Always pair with <label>. Screen readers announce "progress bar, 70 percent" from value and max.
