Code 99: <template>

The <template> element holds HTML that is not rendered when the page loads. JavaScript can clone and insert it into the DOM later. Perfect for client-side templating.

Examples

1. Basic template structure

<template id="card-template">
  <div class="card">
    <h3 class="title"></h3>
    <p class="desc"></p>
    <button class="action">Click</button>
  </div>
</template>
Content inside <template> is inert - no rendering, no scripts run, no images load

2. Cloning with JavaScript

<script>
  const template = document.getElementById('card-template');
  const clone = template.content.cloneNode(true);
  
  clone.querySelector('.title').textContent = 'New Card';
  clone.querySelector('.desc').textContent = 'Created from template';
  
  document.body.appendChild(clone);
</script>

3. Live demo - dynamic user list

4. Template with images - no preload

<template id="img-template">
  <img src="large-photo.jpg" alt="Photo">
</template>
Images inside <template> don't download until cloned and inserted

5. Table row template

<template id="row-template">
  <tr>
    <td class="id"></td>
    <td class="name"></td>
    <td class="status"></td>
  </tr>
</template>
Perfect for dynamically adding table rows without string concatenation

6. Nested templates

<template id="outer">
  <div class="container">
    <template id="inner">
      <span>Nested content</span>
    </template>
  </div>
</template>
Templates can contain other templates. Clone outer, then clone inner separately

Note

template purpose: <template> holds markup fragments that are parsed but not rendered. Think of it as a blueprint for DOM nodes.

Inert content: Inside <template>: scripts don't run, images don't load, forms don't submit, videos don't play. It’s completely dormant.

Access via .content: Use template.content to get a DocumentFragment. Always clone it: cloneNode(true) before inserting.

vs display:none: <div style="display:none"> still loads images and runs scripts. <template> does neither until activated.

vs innerHTML: Templates are faster and safer than building strings with innerHTML. No XSS risk if you use textContent.

Where to put it: Can go in <head> or <body>. Most devs put it at end of <body> near the scripts that use it.

Browser support: All modern browsers. IE11 needs polyfill. Used heavily by Web Components, Lit, and vanilla JS apps.

Checking support: if ('content' in document.createElement('template'))
