Code 100: <title>

The <title> element defines the document's title shown in the browser tab, bookmarks, and search results. Only one per page, required in <head>.

Examples

1. Basic page title

<head>
  <title>HTML 100 Codes Vault</title>
</head>
Shows in browser tab. Keep under 60 characters for Google

2. SEO-optimized title

<title>HTML Template Tag - Clone DOM Elements | HTML 100 Codes</title>
Pattern: Primary Keyword - Secondary | Brand. Front-load keywords

3. Dynamic title with JS

<script>
  document.title = `(${count}) New Messages | Inbox`;
</script>
Updates tab text. Gmail uses this for unread count

4. Title vs h1

ElementWhere it showsPurpose
<title>Browser tab, search resultsSEO, bookmarks
<h1>On the pageContent structure, users
They should match context but don't have to be identical

5. Social media fallback

<title>Learn HTML Template Tag</title>
<meta property="og:title" content="Master HTML <template> in 5 Minutes">
Facebook/Twitter use og:title if present, else falls back to <title>

6. What NOT to do

Bad - keyword stuffing:

<title>HTML HTML Tutorial HTML Learn HTML Code HTML Example</title>
Google penalizes this. Write for humans first

Note

title purpose: <title> is the only required element in <head>. It sets tab text, bookmark name, and search result headline.

Only one allowed: Multiple <title> tags = browser uses the first one. Validators will flag extras.

Length limit: Google displays ~50-60 characters. Longer titles get truncated with ... Put key info first.

No HTML inside: <title>My <b>Bold</b> Page</title> renders the tags as text. Use plain text only.

SEO weight: Highest on-page ranking factor. Match search intent. Unique per page.

Accessibility: Screen readers announce <title> first. Make it descriptive: "Checkout - Cart" not just "Page 3".

Default behavior: If missing, browsers show the filename: code100.html. Looks unprofessional.
