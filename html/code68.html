Code 68: <datalist>

The <datalist> element provides autocomplete suggestions for an <input>. Users can type freely or pick from the list.

Note

datalist rules: Connect with <input list="datalist-id">. The <datalist> id must match. Unlike <select>, user can type values not in the list.

Key behavior: Provides suggestions, not restrictions. Use for autocomplete. Works with text, email, url, range, color, date, time inputs.

Option content: In datalist, only <option value=""> matters. label shows in dropdown. Text content between tags is ignored by most browsers.

select vs datalist: <select> forces choice from list. <datalist> suggests but allows free typing. Choose based on whether custom input is allowed.
