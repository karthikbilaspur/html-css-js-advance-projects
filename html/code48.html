Code 48: <picture>

The <picture> element contains zero or more <source> elements and one <img> element to offer alternative versions of an image for different display/device scenarios.

Examples

Art direction with media queries

Modern image formats with fallback

Resolution switching

Dark mode images


Note

picture rules: <picture> is a container. Browser picks the first <source> with a matching media or type. The <img> is required and acts as fallback + displays the image.

Use cases: Art direction = different crops per screen size. Format switching = AVIF/WebP with JPG fallback. Resolution = 1x/2x for retina. Dark mode = swap images via prefers-color-scheme.

Key difference vs img srcset: img srcset only changes resolution. <picture> can change crop, format, or completely different images.
