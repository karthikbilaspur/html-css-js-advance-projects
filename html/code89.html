Note

div purpose: <div> is the "division" element. It has zero semantic meaning. It’s only for grouping content for styling or scripting.

When to use it: When no other semantic element like <article>, <section>, <nav>, or <aside> makes sense.

Div soup: Avoid excessive nesting. Too many <div> elements makes HTML hard to read and maintain. This is called "div soup".

Block-level: <div> is block-level by default. It starts on a new line and stretches full width. For inline grouping, use <span>.

Accessibility: <div> adds no landmarks or roles. Screen readers skip right over it. If you need a landmark, use a semantic element or add role.

HTML5 rule: Always ask "Can I use something more meaningful?" before reaching for <div>.
