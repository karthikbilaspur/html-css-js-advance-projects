Code 54: <input type="email">

The <input type="email"> element creates a field for entering email addresses. Browsers provide validation and mobile keyboards with @.

Email with autocomplete


Note

email type rules: Browser validates format — must contain @ and domain. Invalid input blocks form submit unless novalidate is on the form.

Key attributes: multiple allows comma-separated emails. pattern enforces regex like .+@example\.com. autocomplete="email" helps browsers autofill.

Mobile UX: type="email" brings up email keyboard with @ and . keys. Better than type="text".

Validation: Checks basic syntax only. Doesn't verify if email exists. Use required to make it mandatory. Use title to explain pattern errors.
