Code 53: <input type="password">

The <input type="password"> element creates a single-line text field where characters are masked. Used for sensitive data like passwords.

Note

password rules: Characters are masked with dots or asterisks. Value is still sent as plain text — always use HTTPS in production.

Key attributes: minlength/maxlength limit length. pattern enforces regex like (?=.*\d) for numbers. required blocks empty submit.

autocomplete: Use autocomplete="current-password" for login forms. Use autocomplete="new-password" for signup/change forms. This helps password managers.

Security: Browser may block autocomplete on password fields if site isn't HTTPS. Never use value="" to prefill passwords. Don't rely on client-side validation only.
